01 / PRODUCT DEFINITION

영상 편집기가 아니라 업로드용 크리에이티브 묶음 생산기

참고 영상 1개를 넣으면, 선택한 포맷에 맞춰 서로 다른 숏폼 홍보 영상 10~30개가 나온다.

UGCLabs의 고객은 편집 자체를 즐기는 사람이 아니다. 이들은 제품, 앱, SaaS, 드랍쉬핑 상품, TikTok Shop 아이템을 빠르게 테스트해야 하는 어필리에이트 마케터다. 이 시장에서는 완벽한 영상 1개보다, 충분히 괜찮은 영상 30개를 빠르게 배포하고 반응을 보는 능력이 더 중요하다.

05 / UNIT ECONOMICS

겉으로는 영상 개수 과금, 내부는 포맷별 원가 관리

UGCLabs는 사용자에게는 단순하게 “월 몇 개 영상”으로 팔지만, 내부 원가는 포맷별로 다르게 관리해야 한다. 스플릿형과 제품 위주형은 보이스오버 중심이라 안정적이고, 인플루언서형과 언박싱은 사람/손/제품 상호작용이 많아 실패율과 원가가 올라간다.

MVP 렌더링 전략

처음에는 Creatomate를 쓴다. 집객 후 Python 렌더러로 갈아탄다.

초기에는 자체 FFmpeg 렌더러보다 Creatomate가 낫다. 5개 포맷의 자막, 레이어, 화면 분할, 오디오 믹싱, 9:16 출력 품질을 빠르게 안정화할 수 있기 때문이다. 집객과 결제 전환이 확인된 뒤, 사용량이 큰 포맷부터 Python + FFmpeg로 이전해 원가를 줄인다.

06 / MARKET POSITIONING

외부 포지션은 크리에이티브 생산성으로 간다

내부 문제의식은 물량과 속도지만, 외부 메시지는 플랫폼과 충돌하지 않는 언어로 설계해야 한다. 결제사, API 제공사, 파트너, 크리에이터에게 오래 갈 수 있는 표현은 “어필리에이트 크리에이티브 생산성”이다.

07 / EXECUTION PLAN

빠르게 출시하려면 랜딩이 아니라 업로드-생성-다운로드 루프를 먼저 만든다

UGCLabs의 첫 출시는 완성형 SaaS가 아니라 판매 가능한 얇은 MVP여야 한다. 목표는 5개 포맷을 모두 완벽하게 만드는 것이 아니라, 한 포맷에서 유료 사용자가 실제로 영상을 생성하고 다운로드하는 흐름을 끝까지 검증하는 것이다.
