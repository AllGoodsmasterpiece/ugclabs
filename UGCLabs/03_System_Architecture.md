# System Architecture

## Standard Pipeline

1. User uploads reference video, product image, and optional model image or account model images.
2. Gemini analyzes the reference video.
3. OpenAI analyzes product image.
4. OpenAI analyzes avatar/model image when provided.
5. GPT creates Product Selling Plan and `generationPrompt`.
6. fal Kling v3 Pro Image-to-Video generates the MP4 with native audio.
7. Server caches the returned MP4 locally for browser playback.

## Important Constraint

The reference video is not passed to fal Kling. It is converted into a text shot plan by Gemini.

## Asset Mapping

With model image:

- `@Element1 = uploaded model image`
- `@Element2 = uploaded product image`

Without model image:

- `@Element1 = uploaded product image`
- creator is generated by prompt and may vary.

## Debug Surfaces

- Reference analysis
- Intent / asset binding
- Asset profiles / product analysis
- Final `generationPrompt`
- Provider errors
